## 背景與現狀 (Context)

目前基礎知識頁面已實作日語語音合成（Web Speech API），並整合了「播放發音」的開關。發音速度目前固定在大約 1.0。為了讓學習過程更具彈性，我們需要讓學習者能自行調整語速。

## 目標與非目標 (Goals / Non-Goals)

**目標 (Goals):**

- 在基礎知識頁面的工具欄中提供一個顯示即時數值的語速滑桿。
- 滑桿應與「播放發音」開關聯動，關閉發音時隱藏滑桿。
- 將語速設定持久化於 `localStorage` 中。
- 點擊卡片發音時能套用當前選定的語速。

**非目標 (Non-Goals):**

- 變更 Web Speech API 的預設語言設定。
- 提供超越語音合成引擎限制的過快或過慢速度。
- 此次變更暫不影響除「基礎知識」以外的其他頁面。

## 關鍵決策 (Decisions)

1.  **UI 組件選擇**: 使用 HTML5 原生的 `input type="range"`，搭配全域設計風格（Sakura 主題色）。
2.  **範圍與間距**: 設定 `min="0.5"`, `max="2.0"`, `step="0.1"`。起始預設值為 `1.0`。
3.  **狀態同步**:
    - 使用 `useState<number>(1.0)` 管理語速。
    - 在 `useEffect` 中初始化 `localStorage.getItem("basics-speech-rate")`。
    - 將 `speed` 參數透過 Props 傳入 `KanaCard` 與 `BasicsCard`。
4.  **連鎖隱藏**: 當 `autoPlay` 為 `false` 時，藉由條件渲染隱藏語速調整區域，以減少介面視覺負荷。

## 風險與權衡 (Risks / Trade-offs)

- **聲音品質**: 語速低於 0.7x 可能會導致語音合成引擎（TTS）產生些微的機器音或破音，但對於辨識片假名構成仍有幫助。
- **介面空間**: 多了滑桿會佔用工具欄空間，但在行動端 (Mobile) 應確保能與 CheckBox 優雅換行排列。
