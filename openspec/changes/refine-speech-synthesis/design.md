## 背景與現狀 (Context)

目前基礎知識頁面的語速滑桿範圍為 `0.5x` 到 `2.0x`。實際測試後發現 `2.0x` 太快且不自然，且五十音的單字發音過於短促、不像真人。

## 目標與非目標 (Goals / Non-Goals)

**目標 (Goals):**

- 優化基礎知識頁面的語速控制範圍，提升實用性。
- 改善單個假名（五十音）發音太過破碎、冰冷的問題，增加尾音的殘響與聽感。

**非目標 (Non-Goals):**

- 修改瀏覽器原生的 Web Speech API 核心引擎。
- 引入複雜的音訊後製或外部 TTS 服務。

## 關鍵決策 (Decisions)

1.  **縮小語速滑桿範圍**: 根據對初學者的實用建議，將 `BasicsPage` 中的 `playbackRate` 滑桿 `max` 值由 `2.0` 降為 `1.2` (間距保持 `0.1`)。
2.  **注入尾音補償 (Decay Injection)**: 修改 `speak` 函式。若發音內容長度小於等於 2（涵蓋所有單一假名及其常見組合），則自動在內容後方附加一個日文頓號 `、`。這能引導引擎在發音結束前延遲關閉，形成更豐沛的尾音聽感。

## 風險與權衡 (Risks / Trade-offs)

- **副作用風險**: 極少數瀏覽器語音可能會因為 `、` 而產生過長的停頓，但根據 Web Speech API 的規範，對於超短字串，這通常能顯著改善其「突然切斷」的問題。
- **權衡**: 犧牲了追求極致高速（1.3x~2.0x）的靈活性，但確保了學習聲音的自然度。
