## 背景 (Context)

目前「基礎知識」頁面 (`/basics`) 提供日語五十音及基礎單字（如數字、時間、日期等）的學習卡片。現有的設計側重於視覺化的假名、羅馬字與翻譯展示，但缺乏聽覺回饋，這對於精確掌握日語發音是至關重要的。

## 目標與非目標 (Goals / Non-Goals)

**目標 (Goals):**

- 整合瀏覽器原生的 **Web Speech API** 提供日語發音。
- 在「基礎知識」頁面的工具欄中增加「自動播放發音」的開關（預設為開啟）。
- 實作全域語音工具函式，具備語速控制、語言選擇等通用功能。
- 在使用者點擊或翻開卡片時觸發語音播放。
- 在卡片角落增加手動播放圖示。

**非目標 (Non-Goals):**

- 此階段不使用 OpenAI 或 Google Cloud 等付費雲端 TTS API。
- 不實作多國語言翻譯發音（僅限日文）。
- 不針對單個單字進行錄製或快取音訊，完全依賴瀏覽器合成。

## 決策 (Decisions)

1.  **實作全域工具函式** (`src/lib/speech.ts`):
    - 封裝 `SpeechSynthesisUtterance`。
    - 鎖定語言為 `ja-JP`。
    - 設定適中的語速（例如 `1.0`），幫助初學者更好地分辨連讀與清音。
2.  **狀態持久化與控制**:
    - 在 `BasicsPage` 組件中使用 `useState` 以及 `useEffect` 來管理 `autoPlay` 狀態（優先從 `localStorage` 讀取並持久化用戶偏好）。
3.  **UI 整合**:
    - 在工具欄中增加與其他模式並排的 Checkbox。
    - 對於 `BasicsCard` 和 `KanaCard` 增加點擊事件監聽，判斷開關狀態決定是否啟動合成。
    - 使用 `lucide-react` 的 `Volume2` 圖示作為手動播放入口。

## 風險與權衡 (Risks / Trade-offs)

- **瀏覽器差異**: 不同作業系統（Windows / macOS / Android / iOS）內建的日語語音包品質差異極大，部分可能聽起來具有機械感。
- **加載時間**: 部分瀏覽器需要在頁面加載後異步加載語音列表，可能導致初次呼叫時出現短暫延遲。
- **隱私權**: 現代瀏覽器通常要求使用者與頁面進行過互動（點擊）後才能播放音訊，這在學習卡片場景中正好符合點擊翻卡的行為，因此風險較小。
